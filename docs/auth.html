When you create a Task, you have the option of making it public or private.

Public tasks are tasks that can be called via an HTTP request and don't require authentication.

Private tasks are tasks that require authentication using a Joined Node token. The token can be provided either using the Authorization request header, or via the token URL query parameter.

Using the Authorization header is recommended as this prevents certain cases where your token can be cached by intermediate HTTP processors.

Unauthenticated requests will result in 403 responses.

Tokens can be presented with the Authorization: Bearer {token} HTTP request header.

Tokens can also be presented with the token={token} URL query parameter
