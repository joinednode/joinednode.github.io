When you create a recipe, you have the option of making it public or private.

  1. Public recipes are recipes that can be called via an HTTP request anytime and don't require authentication.
  2. Private recipes are recipes that require authentication using a Joined Node token. The token can be provided either using the Authorization request header, or via the token URL query parameter.

Private recipes with Unauthenticated requests will result in 403 responses.

There are two ways to pass tokens:

  1. Tokens can be presented with the Authorization: Bearer {token} HTTP request header.
  2. Tokens can also be presented with the token={token} URL query parameter

Creating Tokens

To issue a new token, you just have to make a call to the Joined Node token request endpoint.

Tokens are linked to the container-id you specify when you create a token, and can not be used with any other containers

token.js 

HTTPS POST https://api.joinednode.com/create-token/<your-container-id>
Content-Type: application/json

The request must be authenticated by specifying the token as the token URL query parameter or with the Authorization HTTP request header.

Tokens are only required on recipes that are set to Private but can be used on Public recipes as well.
