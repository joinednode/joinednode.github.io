When you create a Joined Node Recipe, there is a separate section of the page where you can enter encrypted variables, these are then placed in your context object as an array inside context.env and are treated as encrypted parameters. These parameters are stored encrypted when not being executed.


Adding the variables is a simple matter of adding a key as the name, and then the value, in the example above, you can see author is the key and Roger is the value. This could then be access using context.env.author at any point in the Recipe.

hello.js 

module.exports = function(context, callback) { 
	callback(null, {message: "Good morning " + context.data.who + ', sincerely ' + context.env.author + '!' }); 
}

These variables can be useful for storing information such as mongodb urls, twilio information, etc, rather than passing them each time you make the call.

We always encrypt these parameters before saving and only decrypt them either on editing or on execution of the Recipe they are a part of.
