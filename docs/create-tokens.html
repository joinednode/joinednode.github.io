The Joined Node token request endpoint is used to issue a new token. Issuing new tokens is useful when you want to provide your application or specific user with a restricted capability of running tasks.

{% comment %} For example, you can restrict the task token to only execute code from a specific task, or specify secret parameters that will be passed to that code at execution time. {% endcomment %}
token.js 

HTTPS POST https://api.joinednode.com/create-token/<your-container-id>
Content-Type: application/json

The request must be authenticated by specifying the token as the token URL query parameter or with the Authorization HTTP request header.

Tokens are only required on tasks that are set to Private but can be used on Public tasks as well.
