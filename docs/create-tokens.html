The Joined Node token request endpoint is used to issue a new token. Issuing new tokens is useful when you want to provide your application or specific user with a restricted capability of running recipes.

{% comment %} For example, you can restrict the recipe token to only execute code from a specific recipe, or specify secret parameters that will be passed to that code at execution time. {% endcomment %}

Tokens are linked to the container-id you specify when you create a token, and can not be used with any other containers

token.js 

HTTPS POST https://api.joinednode.com/create-token/<your-container-id>
Content-Type: application/json

The request must be authenticated by specifying the token as the token URL query parameter or with the Authorization HTTP request header.

Tokens are only required on recipes that are set to Private but can be used on Public recipes as well.
