Meet Joined Node

At the heart of Joined Node, are Tasks, a Task is a snippet of code that can be called using a simple HTTP request, either directly in a browser or anywhere else. These Tasks can be very simple, or they can be extended as much as you wish. Their major benefits include:

  • Ease of use. No complicated setup, just code.
  • Vastly simplifies / eliminates the need for backend code, boiling it down into reusable, functional pieces.

Joined Node brings you a whole new way to build your code, rather than the traditional, process of setting up servers, building out a full backend, you can instead focus on the frontend and use Joined Node to handle your backend work. All code is executed securely in a containerized sandbox environment, and our quick IDE lets you rapidly build out tasks, allowing you to integrate cleanly with single page web applications, mobile apps, straight HTML apps, etc.

Joined Node Tasks can be used for...

  • Backendless Applications: >Focus on the frontend of your web or mobile applications, and let us handle the server-side.
  • Programmable Webhooks: Run a task on each Twilio message, Stripe Payment, GitHub Push, Twilio message, email received, send email, or any type of webhook you might need, without setting up a single server.
  • Multi Tenant Sandbox: Allow your customers to extend your product using Node.js code instead of creating your own scripting language.

These Joined Node Tasks are also known as microservices, each task does one job, and does it well. This usually means that Tasks tend to avoid interdependencies: if one Task has a hard requirement for other Tasks, then you should ask yourself if it makes sense to make them all part of the same unit. This also helps keep your code clean and organized, while also helping you build out your apps faster when you don't have to worry about huge backend systems.

With Joined Node, we're recreating how you build your apps, rather than building huge monolithic backend services, you can focus on building singular microservices tasks that do one job and do it well. For example you want to send an email, just trigger a post to a task that sends an email with the information included, or if you want to send an SMS message, you can do the same.
